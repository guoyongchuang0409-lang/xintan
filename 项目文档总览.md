## 🎯 使用场景

### 用户端
1. 完成测试 → 自动上传 → 获得分享码
2. 分享分享码给朋友
3. 朋友输入分享码查看结果

### 管理员
1. 访问管理后台
2. 查看所有测试数据
3. 查看统计信息
4. 筛选和管理报告

## 📊 数据流程

```
用户完成测试
    ↓
生成报告和分享码
    ↓
自动上传到 Vercel Postgres
    ↓
存储在数据库中
    ↓
管理员可在后台查看
    ↓
其他用户可通过分享码查看
```

## 🔧 技术栈

### 前端
- Flutter Web
- Provider 状态管理
- Dio HTTP 客户端

### 后端
- Vercel Serverless Functions (Node.js)
- Vercel Postgres (PostgreSQL)

### 部署
- 前端：Netlify
- 后端：Vercel
